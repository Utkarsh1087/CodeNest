# Contributing to CodeNest 🤝

We love your input! We want to make contributing to CodeNest as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Our Development Process

1. **Fork and Clone**: Submitting a pull request? Start by forking the repository and cloning it locally.
2. **Brancing**: Use a clear branch name like `feat-new-feature` or `fix-bug-name`.
3. **Environment**: Use the Provided `.env.example` to set up your local workspace.
4. **Code Quality**: Maintain the "Liquid Glass" design system and ensure TypeScript types are respected.
5. **PR Submissions**: Describe your changes in detail and include screenshots if you've touched the UI.

## Any questions?

Feel free to open an issue or reach out to the project maintainers. Happy coding! 🚀
